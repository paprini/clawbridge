# Setup Guide — clawbridge

## Quick Start (2 minutes)

```bash
# 1. Install
git clone https://github.com/paprini/clawbridge.git
cd clawbridge && npm install

# 2. Run setup agent
npm run setup
```

The setup agent will walk you through everything conversationally.

---

## Setup Modes

### Conversational (recommended)

The setup agent uses any OpenAI-compatible LLM to guide you through configuration. It scans your network, discovers other agents, generates tokens, and tests connections.

```bash
# With OpenAI
OPENAI_API_KEY=sk-... npm run setup

# With Ollama (local)
OPENAI_BASE_URL=http://localhost:11434/v1 OPENAI_MODEL=llama3 npm run setup

# With LM Studio
OPENAI_BASE_URL=http://localhost:1234/v1 OPENAI_MODEL=local-model npm run setup
```

What happens:
1. Agent checks for existing config
2. Scans your local network for other A2A agents
3. Asks you to name this agent (suggests hostname)
4. Helps you connect to discovered peers
5. Generates secure bearer tokens
6. Writes config files
7. Tests connections

### Non-Interactive (power users)

Skip the LLM, answer prompts directly:

```bash
npm run setup:auto
```

Or with flags:

```bash
node src/setup/cli.js --non-interactive
```

### No LLM Available

If no LLM is configured (no OPENAI_API_KEY or OPENAI_BASE_URL), the setup agent automatically falls back to non-interactive mode.

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `OPENAI_API_KEY` | API key for LLM | (none) |
| `OPENAI_BASE_URL` | OpenAI-compatible API URL | `https://api.openai.com/v1` |
| `OPENAI_MODEL` | Model name | `gpt-4o-mini` |
| `A2A_PORT` | Port for A2A server | `9100` |
| `A2A_BIND` | Bind address | `0.0.0.0` |
| `A2A_SHARED_TOKEN` | Shared bearer token | (none — generated by setup) |
| `A2A_CONFIG_DIR` | Config directory path | `./config` |

## What Gets Created

After setup, you'll have three config files:

`config/agent.json` — Your agent's identity:
```json
{
  "id": "my-laptop",
  "name": "my-laptop",
  "description": "A2A agent: my-laptop",
  "url": "http://10.0.1.5:9100/a2a",
  "version": "0.1.0"
}
```

`config/peers.json` — Known peer agents:
```json
{
  "peers": [
    {
      "id": "my-vps",
      "url": "http://10.0.1.10:9100",
      "token": "a2f8...64-char-hex-token"
    }
  ]
}
```

`config/skills.json` — Exposed skills (safe defaults):
```json
{
  "exposed_skills": [
    { "name": "ping", "description": "Health check", "public": true },
    { "name": "get_status", "description": "Agent status", "public": true }
  ]
}
```

## After Setup

Start your A2A agent:

```bash
node src/server.js
```

Test it:

```bash
# Health check
curl http://localhost:9100/health

# Agent Card
curl http://localhost:9100/.well-known/agent-card.json

# Ping (with auth)
curl -X POST http://localhost:9100/a2a \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"jsonrpc":"2.0","method":"message/send","params":{"message":{"kind":"message","messageId":"test","role":"user","parts":[{"kind":"text","text":"ping"}]}},"id":1}'
```

## Troubleshooting

**"No LLM configured"** — Set OPENAI_API_KEY and OPENAI_BASE_URL, or use `--non-interactive`.

**"Authentication failed"** — Check your OPENAI_API_KEY is valid.

**"Model not found"** — Check OPENAI_MODEL matches an available model. For Ollama: `ollama list`.

**"Network scan found nothing"** — Your other agents might be on a different subnet. Enter their IPs manually when prompted.

**"Connection test failed"** — Make sure the peer agent is running (`node src/server.js`) and the token matches on both sides.

**"Port 9100 already in use"** — Another process is using port 9100. Set `A2A_PORT=9101` or kill the existing process.
